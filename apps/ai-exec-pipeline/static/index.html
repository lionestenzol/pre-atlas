Flask spine - claude CLI backend - JSON persistence

What is this?

A shared log where you talk to Claude and track workflow steps. Everything saves to JSON on disk and survives restarts.
  1. 1Chat panel (left): Type a question, hit Send. The Flask server shells out to claude -p "your message" and returns the CLI's reply. No API key needed - uses your logged-in Claude Code session.
  2. 2Pipeline panel (right): Record workflow milestones. Type a step name (e.g. Deploy backend), pick a status, hit Record. Each entry is numbered and timestamped - like a to-do list that logs when things finish.
  3. 3Everything persists: Chat saves to data/chat_history.json, steps save to data/pipeline_state.json. Kill the server and restart - your log is still here.
  4. 4Also an API: GET /chatlog, POST /update_chat, GET /pipeline, POST /pipeline/step. Send X-API-KEY: DEV_KEY header. Other devices (phone, another laptop, a cron job) can hit the same endpoints.
  5. 5Try it now: Type "hello" in the chat box and press Enter. You'll see Claude reply in a few seconds.
