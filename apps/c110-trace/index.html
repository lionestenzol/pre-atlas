What this proves 
PROVES
  PNG works as a *third* substrate channel: telemetry / profiler output.
  Same shape as the program PNG (1 pixel = 1 instruction).
  Hit counts encode losslessly into RGBA (24 bits = 16M hits per cell).
  Hot loops are visually obvious — the body of a fib loop will be ~5× the
  hit count of the setup ops, and that pops as red against the cool tail.

DOES NOT PROVE
  - Anything about "AI verifying execution" (that's pattern 4 territory)
  - Anything about programs running themselves from telemetry
  - Anything about scaling: this is small-program tooling

REAL VALUE
  - Profiler output as PNG → ships through any image-friendly channel
  - Diff two runs by subtracting the trace pixels (visual diff)
  - Stack 100 runs into one PNG for aggregate hot-spot detection
  - Educational visualization: students can SEE the loop body

REAL COST
  - One pass = same overhead as a counter array
  - Encoding to PNG only at HALT, so runtime cost is one image-write
