The Big Picture

Three services. One direction. Everything flows down.

Reads 1,397 conversations. Detects open loops. Calculates how much you're finishing vs. starting. Produces a daily brief telling you what to do.

How They Talk

Services communicate through JSON files on disk. No message queues. No RPC. Python writes cognitive_state.json, TypeScript reads it. Simple, debuggable, zero infrastructure.

Every output is validated against 17 JSON Schema contracts before writing. If the data is wrong, it doesn't write. No silent corruption.

How Routing Works

Five signals feed the mode calculator:

  • Sleep hours — <6 = LOW, 6-7.5 = OK, ≥7.5 = HIGH
  • Open loops — ≥4 = LOW (bad), 2-3 = OK, ≤1 = HIGH (good)
  • Assets shipped — 0 = LOW, 1 = OK, ≥2 = HIGH
  • Deep work blocks — 0 = LOW, 1 = OK, ≥2 = HIGH
  • Money delta — ≤0 = LOW, positive = OK, ≥target = HIGH

Two global overrides that fire from any mode:

  • Sleep LOW → RECOVER (rest overrides everything)
  • Open loops LOW → CLOSURE (finish overrides progression)

This is a lookup table. Not a neural network. Not a prompt. A lookup table. You can read every rule. You can predict every outcome. You cannot sweet-talk a lookup table.

Open Loop Detection

A "loop" is a conversation where you talked about doing something but never said you did it.

The formula: score = user_words + (intent_weight * 30) - (done_weight * 50)

Intent words: want, need, should, plan, going, gonna, start, try, build, create

Done words: did, done, finished, completed, solved, shipped, fixed

High score = lots of talking about doing + no evidence of doing = open loop. The system currently tracks 7 open loops with a closure ratio of 72%.

The Spine

Delta Kernel — The enforcer. Deterministic. Tamper-proof. No negotiation.

What It Does

Reads the Brain's output. Computes your mode using a Markov lookup table. Serves your current state via API on port 3001. Runs a governance daemon with 6 scheduled jobs. Gates every action against mode rules.

  • API: http://localhost:3001/api/state/unified — your complete state in one call
  • CLI: Terminal cockpit showing mode, signals, allowed actions
  • Daemon: Heartbeat (5min), refresh (hourly), mode recalc (15min), day boundaries (6am/10pm)

Hash-Chained Deltas

Every state change is a delta — a JSON Patch with a SHA-256 hash linking to the previous delta. Like a blockchain for your personal state. You can't quietly delete a task or edit your progress. Every mutation is permanent and auditable.

If someone (including you) tampers with the chain, verifyHashChain() catches it. The system is tamper-proof against the person it governs.

Why It Exists

The architecture is the diagnosis.

You start too many things

The open loop detector exists because 1,397 conversations are full of "I'm gonna..." with no "I did." The system counts unfinished sentences against you.

You don't trust yourself to stay honest

That's why routing is a lookup table, not AI. If Claude picked your mode, you could talk your way out of CLOSURE. Math can't be negotiated with.

You generate too many ideas

idea_moratorium: True is in the config. You built a kill switch for new ideas. The Blueprint Generator forces 2-week scope. Max active lanes: 2.

Hash chains are for self-accountability

Every state change is SHA-256 chained. You can't quietly delete a task or edit your streak. The audit log is tamper-proof against the person it governs.

The deferred folder is discipline

IoT, camera feeds, LoRa radio, P2P sync — that's a huge vision. But it's in _deferred/. Not built. Not deleted. Acknowledged and parked. CLOSURE mode applied to the architecture itself.

The system is itself the biggest open loop

40K lines built to solve the problem of building too many things. That's either extraordinary self-awareness or the most sophisticated procrastination ever. Probably both.
